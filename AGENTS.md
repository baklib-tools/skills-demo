# Agent 协作说明

## 仓库定位

本仓库用于向用户**演示**如何安装并使用 [baklib-tools/skills](https://github.com/baklib-tools/skills) 完成各类场景。上游 skills 仓库提供能力与安装方式；本仓库是**演示与示例集合**，不是 skills 本体。

## 分支约定

- **`main`**：仅保留**场景导航**与跨场景说明。以根目录 [README.md](README.md) 为访客与使用者的主入口；不在 `main` 上堆叠各场景的具体实现与冗长文档，避免与场景分支内容重复。
- **场景分支**：**每个使用场景对应一个独立分支**，在该分支中维护该场景的代码、配置、步骤说明等（通常以该分支内的 README 为场景入口）。

## 日常协作提示

- 在 `main` 上工作时，以更新导航与仓库级说明为主。
- 实现、修改或排查某一具体场景时，应切换到**对应场景分支**，并以该分支中的文件为准。
- 若用户未指定场景，可先查看 [README.md](README.md) 中的场景分支列表。

执行 Git 操作前，确认当前在 `main`、工作区无与本次任务无关的未提交混杂变更。若存在远程协作，提交前可视情况先 `git pull` 同步 `main`。

## 新建场景（用户可说「创建新场景」）

当用户要求创建新的演示场景时，按以下顺序执行；**未获得用户对分支命名的明确同意前，不得修改文件、不得提交、不得创建分支**。

1. **命名确认**  
   根据 [README.md](README.md) 中的**分支命名约定**，向用户**明确提议**分支名（例如 `scenario/<slug>`）与一两句场景简述，并**等待用户明确同意**。  
   若 README 尚未列出约定，默认采用：`scenario/<slug>`，其中 `<slug>` 为小写英文或拼音词，单词间用连字符 `-`，避免空格与特殊字符。

2. **更新导航（在 `main` 上）**  
   用户同意后，检出并基于 **`main`** 分支，在 [README.md](README.md) 的**场景分支**表格中新增一行：分支名、场景说明、文档入口（若该场景在分支内有独立 README，可写「见该分支 README」或相对路径说明）。

3. **提交**  
   仅提交本次对 `README.md`（及必要时仅与导航相关的文件）的变更。提交说明建议使用**简体中文**，例如：`文档：新增场景 <slug> 导航`。

4. **创建场景分支**  
   在包含上述提交的 **`main`** 上，创建并切换到新分支（分支名须与用户确认的一致）。后续该场景的实现、文档与资源**只在此分支**进行，勿在 `main` 上堆积场景专属内容。
